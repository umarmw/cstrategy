# Component version strategy

## Aim

Define a proper way how to version components in different stages
- Dev
- Stable
- Release
- Canary

# Basic commands

`npm publish`
`npm version patch`
`npm version minor`
`npm publish --tag beta`
# Pre ID usage

`npm version prerelease --preid=stage`
` v1.1.2 => v1.1.3-stage.0`

`npm publish --tag stage`
